# Proboi Roles

Открытые **роли** для Telegram-бота [Proboi](https://t.me/proboiAI_bot). Роль — это не отдельный бот и не скилл, который ставится всем. Это набор инструкций (персона + задача + база знаний), которые бот «надевает» и после этого **целиком работает как специалист** в этой роли. Подключает роль только тот, кто захочет.

## Что такое роль

Каждая роль — это папка:

```
<role-name>/
├── <role-name>.role.json   — определение: persona, mission, инструменты, триггеры, эскалация
├── knowledge/              — файлы базы знаний (вся рабочая логика и инструкции)
└── README.md              — что делает роль и как её подключить
```

Когда бот работает в роли, он видит **только** её persona/mission и файлы из `knowledge/` — личная память владельца в сессию роли не подмешивается. Поэтому всю предметную логику пишем в `knowledge/`.

## Доступные роли

| Роль | Что делает |
|---|---|
| [`fitness-coach`](fitness-coach/) | Персональный фитнес-тренер: онбординг, КБЖУ, программа тренировок, прогрессия, ведение тренировки, страница прогресса |
| [`finance-advisor`](finance-advisor/) | Личный финансовый помощник: бюджет, подушка безопасности, разбор долгов, накопления под цели, простое инвестирование без хайпа (реалии РФ) |
| [`nutritionist`](nutritionist/) | Нутрициолог: КБЖУ под цель, меню на день/неделю под вкусы и бюджет, разбор продуктов, замены, список покупок, здоровые привычки |
| [`career-coach`](career-coach/) | Карьерный коуч: резюме, сопроводительные письма, подготовка к собеседованию, разбор вакансий, стратегия поиска работы (hh.ru) |
| [`copywriter`](copywriter/) | Копирайтер: готовые тексты для постов, объявлений (Авито), описаний товаров (WB/Ozon), заголовков, рассылок, лендингов — сразу в 2–3 вариантах |
| [`customer-support`](customer-support/) | Поддержка клиентов: готовые ответы на вопросы и жалобы, деэскалация конфликтов, шаблоны и FAQ для владельца малого бизнеса |

## Как бот подключает роль

Владелец подписки (тариф со слотами chat-agent / клон-бота) настраивает агента и подставляет роль: поля из `*.role.json` → в настройки агента, файлы из `knowledge/` → в рабочую папку агента. Дальше бот ведёт себя строго по роли.

## Добавить свою роль

Скопируй структуру существующей роли, опиши `*.role.json` и положи инструкции в `knowledge/`. Минимум кода — максимум понятных инструкций в базе знаний.
