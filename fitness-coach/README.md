# Фитнес-тренер — роль для Telegram-бота Proboi

Это **роль**, а не отдельный бот. Бот Proboi (chat-agent или клон-бот) «надевает» эту роль и полностью работает как персональный фитнес-тренер: онбординг, программа тренировок, прогрессия, ведение тренировки, прогресс-страница.

Ничего не ставится всем пользователям. Роль подключает только тот, кто захочет свой фитнес-бот.

## Что внутри

| Файл | Что это |
|---|---|
| `fitness-coach.role.json` | Определение роли: персона, задача, инструменты, триггеры, эскалация, список файлов знаний |
| `knowledge/coaching.md` | Тренерский плейбук: активация, онбординг, КБЖУ, программа, прогрессия, ведение тренировки (таймер, гифки, итоговая таблица), безопасность |
| `knowledge/progress.md` | Лог тренировок и автозаполнение, калории, прогресс-страница со стриками |
| `knowledge/quotes.md` | Багаж цитат для отдыха между подходами |
| `data/exercises.json` | База из 1094 упражнений с русскими названиями и `gif_url` (CDN ExerciseDB) |

База упражнений теперь **в комплекте**. При активации бот скачивает её на диск одной командой (см. coaching.md §0) и берёт `gif_url` оттуда — гифку техники присылает сразу, без веб-поиска.

## Как бот её подхватывает

1. Владелец подписки настраивает chat-agent (в группе/канале) или клон-бота.
2. Вместо ручного диалога-настройки роли — подставляет эту роль: `persona`/`mission`/`tools`/`triggers`/`escalation` из `fitness-coach.role.json`, а файлы из `knowledge/` кладутся в рабочую папку агента (они и есть «база знаний», которую агент читает).
3. Бот собирает системный промпт из роли и **дальше целиком ведёт себя как тренер** — личная память владельца в эту сессию не подмешивается.

## Инструменты роли

- `send_file` — отправка GIF-демонстраций упражнений и графиков прогресса.
- `container` — генерация графиков (matplotlib) и сборка HTML-страницы прогресса.
- `schedule` — таймеры отдыха между подходами и напоминания о тренировках.

## Статус

v1, обкатка механизма ролей. Тренерская логика отражает правила оригинального бота FitCoach (КБЖУ по Миффлину, линейная прогрессия +2.5/+1.25 кг, деload после 2 неудач, 1RM по Эпли).

**Ограничение таймера отдыха:** живого посекундного отсчёта в одном сообщении нет — роль шлёт старт с цитатой и пинг по готовности через `schedule`. Настоящий тикающий таймер с кнопками (как в оригинальном FitCoach) требует доработки самого бота, не роли.
